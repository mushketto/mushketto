# Технічне завдання (Software Requirement Specification)
**Проєкт:** Інформаційна система для автоматичної генерації тестових завдань на основі інтелектуального аналізу тексту.

## 1. Вступ (Introduction)
### 1.1 Мета
Метою розробки є створення веб-орієнтованої інформаційної системи, яка за допомогою алгоритмів обробки природної мови (NLP) здатна автоматично аналізувати навчальні тексти та генерувати на їх основі тестові завдання різних типів. Це дозволить оптимізувати роботу викладачів та спеціалістів EdTech у процесі створення контрольно-вимірювальних матеріалів.

### 1.2 Межі застосування (Scope)
Система дозволятиме користувачам завантажувати текстові матеріали, налаштовувати параметри генерації (кількість питань, рівень складності, тип питань) та експортувати готові тести у популярні формати (PDF, JSON, формати для імпорту в LMS Moodle). 

### 1.3 Технологічний стек
* **Frontend:** React, TypeScript, Tailwind CSS, Vite.
* **Backend / NLP:** Python (з використанням бібліотек spaCy, NLTK, або трансформаторів Hugging Face).
* **База даних та авторизація:** Supabase.

## 2. Загальний опис (Overall Description)
### 2.1 Перспектива продукту
Система складатиметься з клієнтського веб-додатку для взаємодії з користувачем та серверної частини, яка відповідає за ресурсомісткі обчислення та NLP-аналіз. 

### 2.2 Класи користувачів
* **Викладач/Методист:** Завантажує матеріали, генерує тести, редагує згенеровані питання, зберігає їх у власну базу.
* **Адміністратор:** Керує обліковими записами, моніторить навантаження на систему.

## 3. Функціональні вимоги (System Features)
### 3.1 Модуль авторизації та управління профілем
* Система повинна підтримувати реєстрацію та вхід користувачів (email/password, OAuth через Google).
* Збереження історії згенерованих тестів у профілі користувача.

### 3.2 Модуль завантаження та попередньої обробки тексту
* Підтримка введення тексту вручну або завантаження файлів (TXT, PDF, DOCX).
* Валідація тексту (мінімальна та максимальна кількість символів).
* Очищення тексту від спецсимволів та форматування.

### 3.3 Модуль інтелектуального аналізу (NLP)
* Виділення ключових слів (Keyword Extraction) та іменованих сутностей (NER) з тексту.
* Розпізнавання ключових речень, що містять фактологічну інформацію.

### 3.4 Модуль генерації завдань
Система повинна підтримувати створення наступних типів питань:
* **Multiple Choice Questions (MCQ):** Генерація питання, правильної відповіді та 3-х дистракторів (неправильних варіантів), згенерованих на основі контексту.
* **Fill-in-the-blanks (Заповнення пропусків):** Видалення ключових термінів з важливих речень.
* **True/False (Правильно/Неправильно):** Трансформація фактологічних речень у стверджувальні та хибні форми.

### 3.5 Модуль експорту та інтеграції
* Можливість ручного редагування згенерованих питань перед збереженням.
* Експорт готового тесту у формат PDF (для друку).
* Експорт у форматі XML/GIFT для імпорту в систему Moodle.

## 4. Нефункціональні вимоги (Non-Functional Requirements)
### 4.1 Продуктивність
* Час генерації тесту для тексту обсягом до 5000 слів не повинен перевищувати 15 секунд.
* Інтерфейс (SPA) повинен завантажуватися та бути готовим до роботи за час до 3 секунд.

### 4.2 Безпека
* Усі API запити повинні бути захищені за допомогою JWT токенів.
* Паролі користувачів повинні зберігатися у захешованому вигляді (забезпечується Supabase).

### 4.3 Юзабіліті
* Адаптивний дизайн для коректного відображення на десктопних та мобільних пристроях.
* Інтуїтивно зрозумілий процес генерації, що складається не більше ніж з 3-х кроків (Завантаження тексту -> Налаштування -> Отримання результату).
